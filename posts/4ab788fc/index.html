前端与HTML

1. 什么是前端

定义：前端是构成网页界面、与用户交互的部分

前端技术栈

  • html：设置网页内容
  • css：设置网页样式
  • javascript：设置网页行为
  • http：获取网页

前端应该关注哪些方面

  • 功能：满足用户需求
  • 美观：符合用户审美
  • 无障碍：保证用户可用
  • 安全：保护用户隐私
  • 性能：加速网页运行
  • 兼容：移动端和pc端

前端的边界

  • node.js：是一个后段环境，被广泛应用于前端开发工具的构建，也允许前端开发者使用 JavaScript 构建服务器端应用，是前端和后端的桥梁
  • electron：是一个框架，用于使用前端技术开发跨平台的桌面应用程序，将网页从浏览器应用扩展到桌面平台
  • react：是一个用于构建用户界面的 JavaScript 库，专注于通过组件化的方式构建动态的、交互丰富的 UI
  • webgl：是一个 JavaScript API，允许在网页中直接渲染 3D 图形，将网页从 2D 扩展到了图形和动画等领域

前端的开发环境

  • 运行：edge，chrome等浏览器
  • 编程：vscoed，vim，webstorm等ide

2. HTML

2.1 基础知识

超文本标记语言（HyperText Markup Language,HTML）：是标签语言，而不是编程语言，用于在网页中显示内容，而不是根据输入来输出结果

  • 超文本：文本、图片、链接、表格等
  • 标签：用<>扩起来的，用于标记和设置超文本的属性

基本语法

  • 标签和属性一般使用小写
  • 属性之间一般用空格隔开
  • 属性值一般使用双引号包裹
  • 注释结构：<!-- 注释 -->

2.2 基本结构

html文件的基本结构

  • <!DOCTYPE html>：声明文档遵循HTML5标准
  • 首部（head）：包含文档的元数据，这部分内容不会呈现在页面上
    • <meta>：定义页面的字符编码、作者、描述和关键字等
    • <title>：定义页面的标题，在浏览器的标签页中显示
    • <link>：引入外部资源，常用于引入外部 CSS 文件
    • <script>：定义 JavaScript 代码，也可以引入外部 JavaScript 文件
  • 内容（body）：包含页面的可视内容

实践中通常将script标签放在页面的底部，即先呈现页面内容给用户，再加载页面行为，有利于降低用户视角下的网页加载时间

网页内容的基本结构

  • 头部（header）：位于网页顶部，包含大标题，导航菜单，logo图案等内容
  • 主体（main）：位于网页中心，包含文章、图片、视频等内容
  • 侧边栏（aside）：位于网页两边，包含广告、标签、目录等内容
  • 底部（footer）：位于网页底部，包含版权声明、联系方式、隐私政策等内容

3.1 标题标签

最多支持六级标题

3.2 列表标签

有序列表（ordered list,ol）

无序列表（unordered list,ul）

自定义列表（definition list,dl）

3.3 引用标签

<blockquote></blockquote>：用于表示一段较长的引用文本，是一个块级元素，通常会有缩进或其他样式以便区分引用内容

<q></q>：用于表示一段较短的引用文本，是一个行内元素，浏览器通常会自动为其添加引号来标明引用内容

4. 链接标签

4.1 锚点链接：<a></a>

用于嵌入外部链接，又称为超文本引用（hypertext reference）

  • href：超文本引用（hypertext reference），值为链接目标的URL
  • target：指定链接的打开方式
    • _self：在同一个窗口或标签页打开链接（默认值）
    • _blank：在新窗口或新标签页中打开链接
    • _parent：在父框架中打开链接
    • _top：在整个浏览器窗口中打开链接
  • title：提供链接的额外信息或提示文本，鼠标悬停时显示。
  • rel：定义当前页面与目标页面之间的关系，增强安全性和 SEO 效果
    • noopener：防止新页面访问当前页面的 window 对象，提高安全性
    • noreferrer：不仅防止新页面访问当前页面的 window 对象，还会隐藏引用来源，通常用于增强隐私性
    • nofollow：告诉搜索引擎不要跟踪该链接，用于防止传递链接权重
    • external：指示链接是外部链接
  • download：允许用户点击链接下载指定的文件，并可以选择指定下载的文件名

4.2 图片链接：<img>

用于嵌入静态图像文件，是一个自闭合的标签

  • src：指定图片的来源URL
  • alt：指定图片的替代文本，在图片无法加载时显示
  • width和height：设置图片的显示宽度和高度（单位可以是像素或者百分比）
  • title：指定了鼠标悬停在图片上显示的文本
  • loading：设置图片的加载方式，可以帮助提升网页加载速度
    • lazy：懒加载，只有页面滚动到图片所在位置时才加载
    • eager：立即加载（默认值）
  • srcset：提供不同分辨率或尺寸图片的URL，以适配不同的设备（电脑、手机、平板）
  • sizes：指示图片在不同视口尺寸下的显示尺寸，配合srcset使用

4.3 视频链接：<video></video>

用于嵌入视频文件

  • src：指定视频的URL路径
  • controls：布尔属性，添加后启用视频控件，如播放、暂停、音量调节等
  • autoplay：布尔属性，添加后自动播放
  • muted：布尔属性，添加后静音播放
  • loop：布尔属性，添加后会自动循环播放
  • poster：指定视频加载或播放之前显示的封面
  • preload：是否预加载，有助于增快页面加载
    • auto：自动加载
    • metadata：仅加载视频的元数据
    • none：不预加载
  • width和height：设置视频的宽度和高度
  • <source>标签：提供不同格式的视频源
  • <track>标签：用于添加字幕、说明等轨道信息
  • 标签之间的备用文字：提供浏览器不支持音频播放时的提示信息

4.4 音频链接：<audio></audio>

用于嵌入音频文件

  • src：指定音频的 URL 路径
  • controls：布尔属性，添加后启用音频控件，如播放、暂停、音量调节等
  • autoplay：布尔属性，添加后自动播放
  • muted：布尔属性，添加后静音播放
  • loop：布尔属性，添加后会自动循环播放
  • preload：是否预加载，有助于增快页面加载
    • auto：自动加载
    • metadata：仅加载音频的元数据
    • none：不预加载
  • width 和 height：设置音频播放器的宽度和高度
  • <source>标签：提供不同格式的音频源
  • 标签之间的备用文字：提供浏览器不支持音频播放时的提示信息

4. 表单标签

4.1 输入标签：<input />

用于获取用户输入的数据
type属性：指定输入框的类型

  • text：单行文本输入框
  • password：密码输入框，输入的内容会被隐藏
  • email：邮箱输入框，浏览器会自动验证格式是否有效
  • number：数字输入框，限制用户只能输入数字
  • radio：单选按钮
  • checkbox：复选框
  • button：普通按钮
  • file：文件选择器
  • date：日期输入框
  • time：时间输入框
  • submit：上传按钮，用于提交表单
  • name属性：指定输入字段的名称，表单提交时会使用该名称作为键
  • value属性：指定选型的值
  • maxlength：限制输入框中可输入的最大字符数
  • size：设置输入框的可见字符宽度
  • placeholder：占位符文本，帮助用户理解要输入什么内容
  • required：布尔属性，指定当前输入框为必填
  • readonly：布尔属性，指定当前输入框为只读
  • pattern：指定一个正则表达式，用于验证输入值的格式
  • min和max：指定能够输入的最值

4.2 其他常用标签

<button></button>：用于创建按钮，可以在表单中使用，也可以作为单独的控件，支持更灵活的内容和样式，常用类型有：

  • submit：提交表单
  • reset：用于重置表单所有字段
  • button：用于 javascript 事件

<textarea></textarea>：用于创建多行文本输入框，以接收较长的文本数据，需要指定行和列的长度

<select></select>、<optgroup></optgroup>和<option></option>：第一个标签对定义了一个下拉列表，第二个标签对定义选项分组，第三个标签设置列表中的每个选项
